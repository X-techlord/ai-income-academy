2. The public pages

/ — the hub

Your main landing page: hero, the three course cards, proof wall (recruiter DMs + student results), your instructor bio, sample daily job drops, audience hooks for each demographic (students, graduates, mothers, fathers), the upsell library and FAQ. This is where ads and social posts should point.

/sensei — the showpiece

The cinematic scroll experience: scrolling rotates an AI-generated 360° orbit of you, then two more pinned video chapters present your offers and receipts. Use it as flex material — link it in your TikTok bio, drop it in DMs, show it on live. It doubles as proof that the skills you teach are real ("this page was made with the course's own tools").

/courses/* — the three sales pages

One per course, each with its own accent color, full curriculum accordion (lessons badge-marked Free preview / Standard / Premium), proof, pricing (Standard $49 / Premium $99 — placeholders you can change), FAQ and cross-links to the other courses. The Enroll buttons currently point to a placeholder https://selar.co/ — see section 6.

3. The admin dashboard (/admin)

Your control room for course content. Open /admin and you'll see:

  • The publishing guide — 7 steps from recorded video to live lesson (same flow as section 5 below).
  • The lesson editor — three tabs (one per course). Every module and lesson from your landing pages is pre-loaded. For each lesson:
    • Paste the YouTube URL or ID — the field turns green when a valid ID is captured (it auto-extracts the ID from any YouTube URL format).
    • Set the tier: Free preview, Standard, or Premium.
    • Hit ▶ to preview the actual embed in a popup, so you know the video plays before publishing.
  • Autosave — every keystroke saves a draft to your browser. Close the tab, come back next week, your work is still there (on the same browser/computer).
  • Download course-content.json — exports the finished file. This is the "publish" artifact — see section 5 for what to do with it.
  • Reset to published file — throws away your local draft and reloads whatever is currently live.
⚠️ The admin page is hidden (not linked anywhere public, blocked from search engines) but it is not password-protected — it can't change the live site by itself (only you can, by deploying), but don't share the URL.

4. The student classroom (/classroom)

Where students actually watch. It reads data/course-content.json (the file your admin dashboard generates) and renders:

  • A tab per course, in that course's accent color.
  • Every module as a collapsible section showing how many lessons are live (e.g. "3/6 live").
  • Lessons with a video show ▶ and play in the big player (embedded YouTube, no-cookie player, minimal branding). Lessons without a video yet show 🔒 and a "coming soon" message.
  • Tier badges (free / standard / premium) on every lesson so students see what their plan includes.

How students get here: you send them the link after they pay on Selar (section 6). The page isn't linked from any public page and is blocked from search engines.

5. Publishing lessons — YouTube → live

  1. Record and edit a lesson video.
  2. Upload to YouTube as Unlisted with Allow embedding ON.
    ⚠️ Not Private — private videos refuse to play when embedded, students would see an error. Unlisted = invisible on YouTube (no search, no channel page, no recommendations) but plays anywhere you place the link. This is the correct setting for course videos.
  3. Copy the video link (e.g. https://youtu.be/AbCdEfGhIjK).
  4. Open /admin → pick the course tab → paste the link into that lesson's field (turns green) → set tier → optionally preview with ▶.
  5. Click Download course-content.json.
  6. In the site folder on your computer (AI course/ai-income-academy), replace data/course-content.json with the downloaded file.
  7. Deploy (section 7). Done — /classroom now plays the new lessons.

6. Payments & giving students access

  1. Create your products on Selar — one product per course (or per tier, e.g. "AI Ads — Premium"). Selar handles cards, bank transfer and local payment methods, and pays you out.
  2. Replace the placeholder checkout links. Each course page has two enroll buttons pointing to https://selar.co/. Open courses/ai-ads.html, courses/ai-agents.html, courses/ai-websites.html, search for TODO, and paste your real Selar product URLs. Deploy.
  3. Deliver access after payment. In Selar, set the product's "content" / thank-you message to include the classroom link: https://ai-income-academy.vercel.app/classroom plus a welcome note. Selar shows it to buyers automatically after successful payment.
  4. Know the current limitation: this site is static — the classroom link is secret, but anyone who has it can open it. That's a normal, acceptable launch setup (same as sharing unlisted YouTube links). When revenue justifies it, upgrade to real accounts:
    • Easiest: deliver videos through Selar's own course-hosting and keep this site purely as marketing.
    • Add-on auth: Memberstack or Outseta — paste-in login/paywall for static sites.
    • Full custom: a proper backend (Supabase/Firebase) — ask me when you're ready and I'll build it.

The site lives in three places that stay in sync:

  • GitHub: github.com/X-techlord/ai-income-academy — the backup / source of truth.

  • git commit -m "describe the change"
  • git push
  • vercel deploy --prod

8. The AI assets — how they were made

Every visual was generated through the Arcads.ai API (scripts live in your AIS-OS/scripts/arcads/ folder):

  • Hero images (hub + 3 course pages): GPT Image 2, 16:9.
  • Sensei videos (orbit / builder / closer): Seedance 2.0, 8s, 1080p, audio off — compressed to under 1 MB each for fast loading, orbit encoded with dense keyframes so scrubbing is smooth.
  • Instructor portrait (hub bio): GPT Image 2, 3:4.
  • Identity consistency: your character sheet (AI course/AI course Assets/Tutor-character-sheet.png) is uploaded as a reference image on every generation — it is never displayed on the site itself, exactly so the face stays consistent across all images and videos.

To generate new assets the same way, ask Claude Code — e.g. "generate a new Sensei clip of [scene] with the character sheet as reference and add it to the sensei page." Generation prompts never include your name or company names.

9. Troubleshooting

A page looks like plain black-on-white text with blue links

You opened the .html file directly from the folder. Use the live URL instead — styling, fonts, videos and course data only load through the server.

A lesson shows "coming soon" but I added the video

Adding it in /admin only saves a draft in your browser. You must export the JSON, replace data/course-content.json in the folder, and deploy (sections 5–7).

A video won't play in the classroom ("Video unavailable")

The YouTube video is either Private (change to Unlisted) or has embedding disabled (YouTube Studio → that video → Details → show more → License and distribution → tick "Allow embedding").

I lost my admin draft

Drafts live in the browser you used. Use the same computer/browser, or re-import by opening /admin — it always starts from the live published file if no draft exists.

The orbit doesn't scrub on my phone

It needs one touch on the page first (a browser autoplay rule) — after the first tap, scrolling scrubs normally. This is already handled in the code.

I want to change prices, copy, or add a course

Prices and copy are plain text in the HTML files — edit and deploy, or ask Claude Code. A new course needs: a landing page in courses/, a card on the hub, and an entry in data/course-content.json — a 15-minute job for Claude Code.
