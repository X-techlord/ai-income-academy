How to publish your course

From finished recordings to lessons playing on this site — the full workflow:

Record & edit your lessons

One video per lesson (the curriculum below mirrors your landing pages). Keep filenames matching the lesson titles so you don't mix them up — e.g. M1-S1 Introduction.mp4.

Upload each video to YouTube as Unlisted

  • YouTube Studio → Create → Upload videos.
  • Visibility: Unlisted — anyone with the link can watch, but the video never appears in search, your channel page, or recommendations.
  • ⚠️ Do NOT use "Private" — private videos can only be watched by Google accounts you invite one-by-one, so they will show an error when embedded on this site. "Unlisted" is the correct setting for embedded course lessons.
  • Under "License and distribution" leave Allow embedding ON (it's what lets the video play inside your classroom page).
  • Optional hardening: disable comments, hide like counts, and add a watermark with your brand.

Copy each video's ID

From a URL like https://youtu.be/dQw4w9WgXcQ or https://www.youtube.com/watch?v=dQw4w9WgXcQ, the ID is dQw4w9WgXcQ — the 11 characters after v= or the last path segment. You can also paste the full URL below; the dashboard extracts the ID automatically.

Paste the IDs into the editor below

Pick the course tab, paste each lesson's video URL/ID, set its tier (Free preview / Standard / Premium), and hit the ▶ button to preview the embed right here. Your work saves to this browser automatically as you type.

Export the content file

Click Download course-content.json. Replace the file at data/course-content.json in the site folder with the downloaded one.

Redeploy

  • Commit and push: git add data/course-content.json && git commit -m "update lessons" && git push
  • Deploy: vercel deploy --prod (or connect the GitHub repo in your Vercel dashboard once, and every push auto-deploys).
  • The /classroom page reads this file and renders every lesson with its player.

Gate access for paying students

A static site can't do real logins. The simple launch flow: keep /classroom unlisted (don't link it publicly), and have Selar's post-payment redirect / thank-you email deliver the classroom link plus an access code. When you outgrow that, upgrade to a real gate:

  • Easiest: host lessons on Selar's own course delivery and keep this site as the marketing front.
  • More control: add Memberstack / Outseta to this site (paste-in script, handles login + paywall).
  • Full control: move the classroom to Supabase/Firebase auth — a later project I can build when you're ready.
